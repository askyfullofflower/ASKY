## 2. 后端核心模块

### 2.1 传感器模拟引擎

#### 正常数据池
内置31条基准样本（温度 19–21°C，湿度 48–55%RH），相邻点平滑限速：

```python
NORMAL_TEMP_MAX_STEP = 0.4   # 每 tick 最大温度变化 ±0.4°C
NORMAL_HUM_MAX_STEP  = 1.0   # 每 tick 最大湿度变化 ±1.0%RH
```

#### 历史回填（`prefill_history()`）
回溯 720 个历史点（约 12 分钟），公式：
```
T[i] = 21.5 + 2.0 × sin(i × 0.02) + random[-0.3, 0.3]
H[i] = 52.0 + 5.0 × sin(i × 0.015 + 1) + random[-1, 1]
```

### 2.2 异常注入系统（9 种场景）

| 命令 | 动作 | 效果 |
|------|------|------|
| `inject_fire` | 火灾模拟 | 随机一个库房局部高温，+1.2°C/tick，PM2.5+3.8+tick×0.9 |
| `inject_cold` | 寒潮模拟 | 全馆降温，-1.0°C/tick |
| `inject_rain` | 梅雨模拟 | 全馆高湿，+1.8%RH/tick |
| `inject_dry` | 干燥模拟 | 全馆低湿，-1.8%RH/tick |
| `inject_leak` | 漏水模拟 | 局部渗漏，risk=clamp(45+tick×8, 35, 100)，≥60% 触发 detected=True |
| `inject_gas` | 有害气体 | PM2.5+12.0+tick×1.1，NOx 同步上升 |
| `inject_outlier` | 越界样本 | 按 tick%10 ∈ {2,7} 触发 20% 越界采样 |
| `inject_high_temp` | 高温模拟 | 全馆 +0.5°C/tick |
| `stop_anomaly` | 清除 | 所有异常置 None |

### 2.3 PID-Fuzzy 智能调控算法（`auto_equipment()`）

**动态模糊 Kp**：

```python
# 温度控制
t_error = temp - TEMP_TARGET  # 偏差 = 当前值 - 目标值 20°C
if abs(t_error) > 0.5:
    kp = 25 if abs(t_error) > 3 else 15   # 模糊增益切换
    power = min(100, max(0, int(abs(t_error) * kp)))
    ac.mode = "cooling" if t_error > 0 else "heating"

# 湿度控制
h_error = humidity - HUM_TARGET           # 偏差 vs 目标 50%RH
if h_error > 2.0:
    kp = 10 if h_error > 10 else 5
    dehumidifier.power = min(100, int(h_error * kp))
elif h_error < -2.0:
    humidifier.power  = min(100, int(abs(h_error) * kp))

# 通风协同
main_power = max(ac.power, dehumidifier.power, humidifier.power)
ventilation.power = 80 if main_power > 50 else 30
```

**功率离散化（四档）**：

| 档位 | 触发功率范围 | 实际输出 |
|------|-------------|---------|
| off  | ≤ 0% | 0% |
| low  | 1–40% | 30% |
| medium | 41–75% | 65% |
| high | > 75% | 100% |

|------|---------|---------|---------|---------|
| AC 空调 | heating | cooling | ±0.8°C/tick × (power/100) | ±0.1%RH/tick（微弱） |

| 通风系统 | — | ventilating | ±0.05°C/tick | ±0.1%RH/tick |

### 2.5 告警引擎

#### 告警生命周期

```
[阈值超标检测] → open_alarm_record()
  ↓
  状态：active（新建，未处理）
  ↓  ← 超时 180s 未处理
  状态升级（一般→较高→严重）
  ↓  ← 人工受理
  状态：processing（处理中）
  ↓  ← 人工完成
  状态：resolved（已恢复）
  ↓  ← 可选复盘
  状态：review（待复盘）→ resolved（关闭复盘）
```

#### 告警记录结构

```python
{
    "id": "ALM-20260512103045-0001",
    "alarm_time": "2026-05-12 10:30:45",
    "level": "严重 | 较高 | 一般",
    "location": "档案库房A",
    "reason": "温度 28.5°C 超过上限 24.0°C，偏差 +4.5°C",
    "source": "阈值监测 | 环境安全 | AI预警",
    "status": "active | processing | review | resolved",
    "disposal_result": "待处置",
    "resolved_time": ""
}
```

#### 自动升级机制（`maybe_escalate_unresolved_alarms()`）

- `active` 状态超过 **180 秒** → 升级等级（一般→较高→严重）
- `processing` 状态超过 **360 秒** → 升级，追加 `disposal_result` 历史记录
- `sync_threshold_alarm_records()`：每 tick 扫描 7 库房，key = `threshold:{库房名}:{问题类型}`，自动开启/升级/关闭

### 2.6 飞书 Webhook 通知（`build_feishu_alert_payload()`）

**双通道**：温湿度阈值超标通道 + 环境安全告警通道（气体/漏水/火灾）。

**去重签名**：120s 内相同告警不重复推送。

**内容构建逻辑**：按偏差严重度降序排列库房，最多显示 4 个，超出部分折叠为"等N个库房"。

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/` | 返回 PC 端前端 HTML |
| GET | `/api/alarm-records?limit=200` | 分页获取报警记录 |

### 2.8 broadcast_sensor_data() — 14 步主循环（每 1 秒）

```
1. 采样传感器数据（含平滑限速）
2. 叠加异常效果（inject_* 逻辑）
3. 执行 PID-Fuzzy 调控计算
4. 施加设备物理效应
5. 计算漏水检测状态
6. 计算空气质量（PM2.5 / NOx）
7. 计算各库房独立温湿度
8. 楼层聚合统计（floor_stats）
9. 同步阈值告警记录
10. 检查升级逻辑
11. 飞书通知判定
12. 追加历史记录（MySQL / deque）
13. 线性趋势预测
14. WebSocket 广播到所有连接客户端
```

### 3.6 温湿度场云图（`renderHeatmap()`）— IDW 插值算法

#### 书架探测点模拟（`buildShelfProbeSet()`）

基于平面图推算书架：
- `rackCount = clamp(round(roomWidth/4.6), 3, 6)` 排数
- `pointsPerRack = clamp(round(roomDepth/4.8), 4, 8)` 每排探测点数
- 额外 6 个角落/门口补充点

**每个探测点独立计算温度偏差（8因子叠加）**：

```
tempDelta =
  statusFx.tempBias                    // 状态偏置（temp_alert → +1.3°C）
  + doorFlow × doorTempDrift           // 门口气流影响
  + cornerStagnation × 0.55           // 角落滞留
  + shelfRetention × (-0.42)          // 书架遮挡散热减弱
  + hvacPlume × hvacSign × 0.9        // 空调出风口高斯分布羽流
  + nx × 0.34 - ny × 0.24             // 空间位置梯度
  + aisleWave × 0.24                  // 书架间通道气流波
  + largeWave × 0.68                  // 大尺度对流波动（含时间相位）
  + noise × 0.62                      // 随机噪声
```

湿度偏差使用反向系数（HVAC 出风处湿度反向，角落滞留湿度更高）。

#### IDW 反距离加权插值

```javascript
// 对每个 5×5px 格子
val = Σ(wᵢ × vᵢ) / Σwᵢ   // wᵢ = 1 / dist^1.9
```

power = 1.9（接近平方反比，比 2.0 更平滑）

**颜色映射**：
- 温度：14°C → HSL(240°,蓝) ~ 24°C → HSL(0°,红)
- 湿度：40% → HSL(180°,青) ~ 70% → HSL(250°,深蓝)

## 6. 智能算法详解

### 6.1 线性趋势预测

```python
# 取最近 30 个历史点
recent = history[-30:]
n = len(recent)
x = range(n)
x_mean = (n-1)/2

# 最小二乘线性回归
slope_t = Σ((xᵢ - x_mean) × (tᵢ - t_mean)) / Σ((xᵢ - x_mean)²)

# 外推 1 小时 = 3600 点
next_hour_temp = current_temp + slope_t × 3600

# 趋势分类
temp_trend = "rising"  if slope_t > +0.001
           = "falling" if slope_t < -0.001
           = "stable"  otherwise

# 置信度：基于 R² 系数
```

### 6.2 空气质量模拟

**基线**：PM2.5 基础 15 μg/m³，NOx 基础 0.03 mg/m³，含 ±0.5/±0.005 随机扰动。

**异常叠加**：

| 异常类型 | PM2.5 增量 | NOx 增量 |
|---------|-----------|---------|
| 火灾 | +3.8 + tick×0.9 | +0.05 + tick×0.02 |
| 有害气体 | +12.0 + tick×1.1 | +0.08 + tick×0.025 |
| 无异常 | 0 | 0 |

### 6.3 漏水检测模型

```python
# 正常状态
risk = random(2, 18)
detected = False

# 漏水注入
tick 递增
risk = clamp(45 + tick × 8, 35, 100)
detected = (risk >= 60)
location = random.choice(ARCHIVE_ROOMS).name
```
